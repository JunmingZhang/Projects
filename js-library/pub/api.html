documentation

What can this library do?

This front-end library helps to play slides (for example, course slides) on the webpage, slides can be played automatically with some animations or manually like PowerPoint. With this library, the user can play slides online instead of downloading them and read it locally, they can choose to play slides manually, so they switch slides by clicking the slide, or they can play it automatically like watching a video, they can choose different playing speed and different animations (like horizontal scrolling, vertical transition, fading), and pause automatic transition by hovering, and other fancy actions or additional features help you read slides. For developers, if they want to design a box for playing slides with any possible area, they can import this library to support playing slides in their box. There is no universal way to design many aspects of slide playing, for example, slides animation. With this library, when a developer needs to realize similar functionality, they do not have to spend lots of time on designing slides playing algorithms and write their code. For example, the way to realize fading can be entirely different from scrolling. My library can be different from others because it supports many animations, playing speed, and automatically playing for slides. For instance, more and more instructors choose to post slides and notes online, but most lecture slides posted online can only be switched by scrolled or clicked manually in the browser. With this library, slides can be played online as PowerPoints developed by Microsoft.

In this library, you can use the default properties to build up the framework of the slide, and also use the default animations and filters to show slides. You can also write your properties to play the slides and extend the library. By designing your settings, please code them in a separate CSS file.

Features

  • Two ways to play the slideshow: the user can either play the slides manually or automatically. There are different ways to play slides manually, including clicking the page directly, clicking buttons, and clicking dots, I will introduce the last two later.
  • Two ways to play the slideshow: the user can either play the slides manually or automatically. There are different ways to play slides manually, including clicking the page directly, clicking buttons, and clicking dots, I will introduce the last two later.
  • Order of playing: the user can play slides either forward or backward.
  • Button guidance to play the slides: for playing manually, the developer can either add two buttons so the user can click either button to play the slides back and forth, or add a set of dots, so the user can jump to a page by clicking a dot, and if the developer enables hovering here, the user can hover at the dot to preview a slide. For playing slides automatically, the developer can either add two buttons so the user can click either button to pause the slides permanently (until the user unlocks the slide) or play the slides permanently (until the user locks the slide). And the dots for automatic playing have the same functionality as playing manually.
  • Gesture: a set of events to play the slides. Include:
    • Click: if being enabled, for manually playing, the user can click the slide to move to the next or previous page. For automatically playing, the user can click the slide to lock the slide (so not pause the slides) until clicking or double-clicking the slide.
    • Double click: only used in automatically playing, if being enabled, the user can double click the slide to unlock the slide permanently (so the user cannot pause the slide even if hover on the slide), the user can delete the state permanently unlocking by clicking, or double-clicking another time.
    • Hover: if this is enabled, for using dots to show slides, the user can hover at one dot to preview a page, and the slide show will be paused. For automatically playing, the user can hover on the page to pause playing. Hover can pause the page, but not lock or unlock the page.
  • Animation: there are different animations for the user to transit between pages, my implemented animations are in the file JSlide_animation.css, the developer can also choose to implement and add his animation by implementing a new CSS animation style in a CSS file.
  • Filter: there are different filters for slides, which provide different views, my implemented filters are in the file JSlide_filter.css, the developer can also choose to implement and add his filter by implementing a new CSS filter style in a CSS file.
  • Caption and page number: the developer can show the caption and page number for each slide for users to read more conveniently.

Components

  • JSlide.js: function implementations of the library, you can import the functions in this library for actions of building the slideshow.
  • JSlide_properties.css: the default properties and settings of the slide show built with this library.
  • JS_animation.css: the defualt animations of the slide show in this library.
  • JS_filter: the defualt filters of the slide show in this library.

Use the guidance function "easySetup"

If you want to use the function easySetup, you only have to consider setting arguments, but not the order of method execution. The CSS and the HTML code snippets are the same as above. In this subsection, we use the methods defaultArgs and easySetup to generate the example above. 

            
            test = new JSlide()
            args = test.defaultArgs("#easySetup", "slide_box", "slide", "flex")
            args.clickable = true
            args.showCaption = true
            args.showPageNumber = true
            args.pageNumberClass = "pagenumber"
            args.captions = captions
            args.captionBoxClass = "caption_box"
            args.captionClass = "caption"
            
            test.easySetup(args)

Arguments of the library & Function executions

Here we talk about the arguments used in the library and the order of library methods executions. I use the body of the function defaultArgs and easySetup as example. I will introduce the library methods and user-accessible parameters in the next section. 

            
            defaultArgs: function(id, slideBoxClass, slideClass, slideDisplayClass) {
                var args = {
                    id: id,
                    slideBoxClass: slideBoxClass,
                    slideClass: slideClass,
                    slideDisplayClass: slideDisplayClass,
                    image_path: null,
                    showPageNumber: false,
                    pageNumberClass: null,
                    showCaption: false,
                    captions: null,
                    captionBoxClass: null,
                    captionClass: null,
                    clickable: false,
                    useButton: false,
                    buttonBoxClass: null,
                    buttonClass: null,
                    hoverable: false,
                    useDot: false,
                    dotsBoxClass: null,
                    dotClass: null,
                    dotActiveClass: null,
                    auto: false,
                    reverse: false,
                    dblclickable: false,
                    speed: 1,
                    animation: null,
                    filter: null
                }

                return args
            }                
            
         
            
            easySetup: function(args) {
                this.link()
                this.selectDisplayArea(args.id, args.slideBoxClass, args.slideClass, args.slideDisplayClass)
            
                if (args.image_path) {
                    this.buildSlideBox()
                    this.upload_pictures(args.image_path)
                }
                this.load_pictures()
            
                this.setAuto(args.auto) // if display automatically
                this.setReverse(args.reverse) // show slides reversely
            
                // page number and captions
                this.setShowPageNumber(args.showPageNumber, args.pageNumberClass)
                this.addPageNumber()
                this.setShowCaption(args.showCaption, args.captionBoxClass, args.captionClass)
                this.addCaption(args.captions)
            
                // gesture
                this.setClickable(args.clickable) // click
                this.setHoverable(args.hoverable) // hoverable
                if (args.auto) {
                    this.setDblclickable(args.dblclickable) // double click
                }
            
                // display method
                this.setButton(args.useButton, args.buttonBoxClass, args.buttonClass) // buttons
                this.setDots(args.useDot, args.dotsBoxClass, args.dotClass, args.dotActiveClass) // dots
            
                // add animation
                if (args.animation) {
                    this.addAnimation(args.animation)
                }
            
                // add filter
                if (args.filter) {
                    this.addFilter(args.filter)
                }
            
                this.showSlides() //show slides
                if (args.auto) { // shows slides automatically
                    this.setSpeed(args.speed) // transition speed for each slide
                    this.showSlidesAutomatically()
                    if (args.useButton) { // enable buttons for automatical slideshow
                        this.showSlidesByButtonClickAuto()
                    }
                    if (args.useDot) { // enable dots for automatical slideshow
                        this.showSlidesByDotAuto()
                    }
                } else { // show slides manually
                    if (args.clickable) { // click to turn over the page
                        this.showSlidesByClick()
                    }
                    if (args.useButton) { // use button to turn over the page
                        this.showSlidesByButtonClick()
                    }
                    if (args.useDot) { // use dot to turn over the page
                        this.showSlidesByDotClick()
                    }
                }
            },

User-accessible Parameters

These are detailed introduction of parameters necessary for the user to create the JSlide object, but the user does not need to set them by directly accessing, since there are library functions provided for users to set these parameters.

parameter type default (if required, labelled *) description
container HTML component <div> * The location of the slides in the page or example box. This is captured by the string id of this HTML component. Functions to capture this component will be introduced later.
slideBoxClass String * The name of the CSS style (properties) of the slide box. There is a default one in JSlide_properties, you can also write your own slide box style.
slideClass String * The name of the CSS style (properties) of the slide page. There is a default one in JSlide_properties, you can also write your own slide page style.
slideDisplayClass String * The display behavior of an element, such as "block", "flex". Please read CSS display Property for more details. I recommend "flex" or "block" here.
image_path Array (JS object) or null null A list of paths to the slides or images.
showPageNumber Boolean false If the page number of slides is shown.
pageNumberClass string null The name of the CSS style (properties) of the page number class. There is a default one in JSlide_properties, you can also write your own page number style.
showCaption Boolean false If the caption of each slide is shown.
captions Array (JS object) or null null A list of captions for each page of slide. The number of captions must be equal to the number of slides.
captionBoxClass string null The name of the CSS style (properties) which defines the location of the caption, the caption text alignment, and the display method. There is a default one in JSlide_properties, you can also write your own caption box style.
captionClass string null The name of the CSS style (properties) of the caption letters. There is a default one in JSlide_properties, you can also write your own caption style.
clickable Boolean false If true,
  • for playing manually, this enables the reader to read another page.
  • for playing automatically, this enables the reader to lock the slides player and pause for playing, and so the reader cannot restart playing automatically by leaving the mouse from the slide, until the reader click another time or double click.
useButton Boolean false If true,
  • for playing manually, this enables the reader to read the previous or next page by two buttons.
  • for playing automatically, this enables the reader to lock the slides player and pause for playing (like clickable above), or invalidate pausing by hovering by two buttons.
buttonBoxClass string null The name of the CSS style (properties) which defines the location of two buttons, the buttons alignment, and the display method. There is a default one in JSlide_properties, you can also write your own button box style.
buttonClass string null The name of the CSS style (properties) of the button view. There is a default one in JSlide_properties, you can also write your own button view style.
hoverable Boolean false This works for playing automatically. If true,
  • for hovering the cursor over the slide, this enables the reader to pause for playing slides temporarily. After the reader leaves the cursor from the slide, the automatic playing restarts.
  • for hovering the cursor over a dot, the slide view jumps to the slide corresponding to the dot hovered. After the reader leaves the cursor from the dot, the slide view jumps back to the original view.
useDot Boolean false If the reader use dots to read each slide. The reader can jump to the slide by clicking the corresponding dot.
dotsBoxClass string null The name of the CSS style (properties) which defines the location of dots, the dots alignment, and the display method. There is a default one in JSlide_properties, you can also write your own dots box style.
dotClass string null The name of the CSS style (properties) of the dot view when the dot is not active (the corresponding slide is not shown now). There is a default one in JSlide_properties, you can also write your own dot view style.
dotActiveClass string null The name of the CSS style (properties) of the dot view when the dot is active (the corresponding slide is shown now). There is a default one in JSlide_properties, you can also write your own dot view style.
auto Boolean false If the slides are shown automatically.
reverse Boolean false If the slides are played reversely (both manually and automatically).
dblclickable Boolean false This works for playing automatically. If true, pausing by hovering the cursor over the slide is invalidated. The invalidation can be cancelled by double clicking again, or click the slide.
speed number 1 The transition duration for each slide page (unit: second).
animation string null The animation effect for slides transition, default animations implemented in JSlide_animation including:
  • fade.
  • move_vertical: move vertically (forwardly or reversely).
  • move_horizontal: move horizontally (forwardly or reversely).
The programmer can also design their own animation.
filter string null The filter effect for slides transition, default filters implemented in JSlide_filter, including:
  • blur.
  • brightness.
  • contrast.
  • grayscale.
  • hue_rotation.
  • sepia.
  • invert.
  • transparent.
  • saturate.
The programmer can also design their own filter.

user-accessible library methods

Setup slideshow with raw library functions

  • link: link the web page (document object) to the JSlide instance.
  • selectDisplayArea: select the framework of slides with the given id. If the slide box has been hard-coded, it will select the box of slides as well. Asign the CSS properties slideBoxClass, slideClass, and the display method slideDisplayClass to the JSlide object.
    Parameters:
    • id: string, the id of the slides framework.
    • slideBoxClass: string, the class name of the slide box in the CSS stylesheet.
    • slideClass: string, the class name of the slide box in the CSS stylesheet.
    • slideDisplayClass: string, the display method of the slide.
  • buildSlideBox: generate the slide box if it is not hard-coded in the HTML file.
  • upload_pictures: link the paths of slides to the JSlide object, call this function when slide paths are not hard-coded in the HTML file.
    Parameters:
    • image_path: array, the list of paths to slides.
  • load_pictures: push HTML components of slides to the storage of the JS object.
  • setAuto: enable or disable automatic slideshow.
    Parameters:
    • auto: Boolean, if the slides are shown automatically.
  • setReverse: play the slides forwardly or reversely.
    Parameters:
    • reverse: Boolean, if the slides are shown in forward or reverse order.
  • setShowPageNumber: choose whether to show the page number or not.
    Parameters:
    • showPageNumber: Boolean, if the page number is shown.
  • addPageNumber: add the page number to the slideshow.
  • setShowCaption: choose whether to show the captions or not.
    Parameters:
    • showCaption: Boolean, if captions are shown.
    • captionBoxClass: string, the class name of the caption box in the CSS stylesheet.
    • captionClass: string, the class name of the caption in the CSS stylesheet.
  • addCaption: add the captions of each slide to the slideshow.
    • captions: array, a list of captions for each slide.
  • setClickable: set to activate or inactivate the click gesture during the slideshow.
    Parameters:
    • clickable: if the click gesture is active during the slideshow.
  • setHoverable: set to activate or inactivate the hover gesture during the slideshow.
    Parameters:
    • clickable: if the hover gesture is active during the slideshow.
  • setDblclickable: set to activate or inactivate the double click gesture during the slideshow.
    Parameters:
    • dblclickable: if the double click gesture is active during the slideshow.
  • addAnimation: enable and add the transition animation to the slideshow.
    Parameters:
    • animation: the name of the animation.
  • addFilter: enable and add the filter to the slideshow.
    Parameters:
    • filter: the name of the filter.
  • setSpeed: set the transition duration for the transition animation.
    Parameters:
    • speed: the transition duration for the transition animation.
  • showSlides: add filters and animations to the slideshow if enabled, also initiate the slideshow.
  • showSlidesAutomatically: if the reader enables automatic slideshow, start showing the slides automatically, also add click, double click and hover event listeners if enabled.
  • showSlidesByClick: use this function to show slides if the click getsure is enabled and the reader chooses to show slides manually, if this function is executed, the reader can turn each slide over by clicking the slide in the slideshow.
  • showSlidesByButtonClick: use this function to show slides if buttons are added and the reader chooses to show slides manually, if this function is executed, the reader can turn each slide over forward and backward by clicking buttons added.
  • showSlidesByButtonClickAuto: use this function to show slides if buttons are added and the reader chooses to show slide automatically, if this function is executed, the reader can lock the slide so the reader cannot start playing the slideshow if the reader leaves the cursor from the slide, or disable pausing by hovering, by clicking buttons added. The action added by either button can be cancelled by clicking either button.
  • showSlidesByDotClick: use this function to show slides if dots are added and the reader chooses to show slides manually, if this function is executed, the reader can choose to read some slide by clicking the corresponding dot.
  • showSlidesByDotAuto: use this function to show slides if dots are added and the reader chooses to show slides automatically, if this function is executed, the reader can choose to read some slide by clicking the corresponding dot.

Setup slideshow with easySetup

Please read the getting started section to see how to use the easySetup function to build your slides conveniently.

  • defaultArgs: generate all parameters needed to set up the slideshow. The required arguments must be input into this function, other parameters have their default values.
    Parameters:
    • id: string, the id selector of the framework of slides.
    • slideBoxClass: string, the CSS properties class for the slide box.
    • slideClass: string, the CSS properties class for the slides.
    • slideDisplayClass: string, the display method for each slide, like flex and block.
  • easySetup: a guidance function to setup the slideshow in one step, after the parameters are set (default arguments returned by defaultArgs).
    Parameters:
    • args: javascript object, a key-value pair returned by defaultArgs, the initial returned values are all default arguments, you can update arguments as you need to build up the slideshow.
